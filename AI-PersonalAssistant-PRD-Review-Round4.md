## 🟢 建议改进

### R27. 架构图中 LiteLLM 引用需清除

**位置**：第 292 行

**问题**：架构图 AI 引擎层仍写 `LLM Router (LiteLLM / 自研)`，但 LiteLLM 是 Python 库，与 Rust 后端不兼容。技术栈选型表 (5.2) 已明确写"自研 Rust 路由器"。

**建议**：改为 `LLM Router (自研 Rust 路由器)`。

---

### R28. 功能清单缺少 v1.0/v2.0 范围标注

**位置**：第三章核心功能设计、3.3.2 第三方集成清单

**问题**：功能设计章节描述了完整愿景（知识图谱、风险预测、PPT 报表、Asana/Linear 集成等），但没有标注哪些是 v1.0 范围、哪些是 v2.0。读者可能误以为所有功能都在 v1.0 交付。

Timeline 已明确 v1.0 缩减范围（砍掉知识图谱自动构建、风险预测、高级报表），但功能描述章节没有同步。

**建议**：在第三章开头或每个子功能后标注版本范围：
- `[v1.0]` 项目态势感知、会议助手、基础报表（周报/月报）
- `[v1.0]` Jira + 飞书集成
- `[v2.0]` 知识图谱自动构建、风险预测模型、PPT/Word 报表
- `[v2.0]` Asana/Linear/Teams/Confluence 集成

---

### R29. 第三方集成清单过于庞大

**位置**：第 214-220 行（3.3.2 第三方集成清单）

**问题**：清单列出了 7 个类别、20+ 个平台，但 v1.0 实际只做 Jira + 飞书。这份清单给读者的预期与实际交付差距过大。

**建议**：拆分为两个表格：
- **v1.0 支持**：Jira、飞书（项目/日历/文档）、GitHub
- **v2.0 规划**：Asana、Linear、Slack、Teams、Notion、Confluence 等

---

### R30. 章节编号不规范

**位置**：第十章之后

**问题**：第十章之后使用了 `十-2`、`十-3`、`十-4`、`十-5` 这种非标准编号，应使用正式的章节编号。

**建议**：
- 十-2 → 十一
- 十-3 → 十二
- 十-4 → 十三
- 十-5 → 十四
- 总结 → 十五
- 附录 A 保持不变

---

### R31. 存在重复分隔线

**位置**：第 1047-1049 行

**问题**：总结章节结束后有两个连续的 `---` 分隔线，应为一个。

```markdown
---

---

## 附录 A
```

**建议**：删除多余的 `---`。

---

**建议**：在文档开头（第一章之前）添加目录：

### R33. 代码示例中存在 Python 库引用

**位置**：第 307 行

**问题**：架构图专用模型层写 `图表生成 (Code Interpreter + Matplotlib)`，Matplotlib 是 Python 库。虽然实际部署为独立服务，但在 Rust 架构文档中引用 Python 库名容易混淆。

**建议**：改为 `图表生成 (独立 Python 服务 / ECharts 前端渲染)`。
